= CppTest Utility
:toc: macro
:source-highlighter: highlight.js

CppTest is a command-line tool designed for competitive programming (CP). 
It automates compilation, testing, test generation, and stress testing, focusing on a clean workflow and secure execution isolation.

toc::[]

== Features

*   **Test Generation:** Automatically compiles `gen.cpp` and generates inputs based on configuration.
*   **Stress Testing:** Runs a brute-force comparison (`main` vs `slow`) on generated tests until a mismatch is found.
*   **Memory Profiling:** Automatically measures and reports peak RAM usage for each test.
*   **Colored Diff:** Uses `git diff --word-diff` to precisely highlight character-level mismatches on Wrong Answer (WA).
*   **Interactive Mode:** Allows manual terminal input for interactive problems.
*   **Isolation:** By default, tests run in `/tmp`, keeping the project folder clean from build artifacts.
*   **Static Analysis & Auto-Fix:** Integrated `clang-tidy` and `clang-format` checks with the ability to automatically fix code formatting.
*   **Custom Checkers:** Full support for `check.cpp` in problems with multiple valid outputs.

== Installation

To install or update the utility, run this single command in your terminal:

[source,bash]
----
rm -rf /tmp/cpptest_inst && git clone --depth=1 https://github.com/MootSasa/CppTest.git /tmp/cpptest_inst && bash /tmp/cpptest_inst/install.sh && rm -rf /tmp/cpptest_inst
----

== Project Structure

CppTest works best when the problem directory is organized as follows:

[source,text]
----
TaskA/                      <-- Root folder of the problem
├── main.cpp                <-- [REQUIRED] Your optimized solution
│
├── tests/                  <-- [OPTIONAL] Manual tests (downloaded from contest)
│   ├── 1.in
│   ├── 1.out               <-- Expected output (optional)
│   └── 2.in
│
├── gen.cpp                 <-- [OPTIONAL] Test Generator
├── tests.cfg               <-- [OPTIONAL] Generator configuration
├── gen_tests/              <-- [AUTO] Folder where CppTest saves generated tests
│
├── check.cpp               <-- [OPTIONAL] Custom Checker (for tasks with multiple answers)
└── slow.cpp                <-- [OPTIONAL] Brute-force solution (for Stress Testing)
----

== Usage Guide

=== 1. Basic Testing
Simply run the command in the problem folder. It compiles `main.cpp` and runs it against all found tests, displaying execution time and memory usage.

[source,bash]
----
cpptest
----

=== 2. Interactive Mode (`-I`)
Used for interactive problems. It compiles the solution and attaches it to standard I/O so you can manually type inputs and verify responses.

[source,bash]
----
cpptest -I
----

=== 3. Using Generator (`gen.cpp`)
If `gen.cpp` exists, the utility will:
1.  Compile it.
2.  Read `tests.cfg` (if present) or use defaults.
3.  Generate inputs into `gen_tests/`.
4.  Run `main.cpp` against them.

**Config format (`tests.cfg`):** `Count Arguments`
[source,text]
----
10  1 100    # Run ./gen 1 100 (10 times)
5   500 1000 # Run ./gen 500 1000 (5 times)
----

=== 4. Stress Testing (`-S`)
Used to find bugs by comparing your optimized solution (`main.cpp`) against a slow but correct one (`slow.cpp`).
Requires: `main.cpp`, `slow.cpp`, `gen.cpp`.

[source,bash]
----
cpptest -S
----

== Command Line Flags

[cols="1,4", options="header"]
|===
| Flag | Description

| **Execution** |
| `-I`, `--interactive` | Run the solution interactively in the terminal. No automated tests are executed.
| `-S`, `--stress` | Run infinite comparison test (`main` vs `slow`).

| **Test Selection** |
| `-a`, `--manual` | Run ONLY manual tests (from `tests/` and root).
| `-g`, `--generated` | Run ONLY generated tests (from `gen_tests/`).
| `filename.in` | Run ONLY specific file (e.g., `cpptest tests/1.in`).

| **Generation** |
| `-n`, `--no-gen` | Skip test generation. Use existing tests in `gen_tests/`.
| `-G`, `--gen-only` | Generate tests and exit. Do not run solution.
| `--no-gen-config` | Ignore `tests.cfg` and use defaults (100 tests).

| **Code Modifiers** |
| `-F`, `--fix` | Auto-Fix formatting and linter errors. **Modifies file and exits.**
| `-R`, `--no-comments`| Permanently removes `//` and `/* */` from `main.cpp` and **exits**.

| **Advanced** |
| `-i`, `--in-place` | Run directly in source directory (disable `/tmp` isolation).
| `-f`, `--fast` | Fast run. Skip static analysis (Linters).
| `-K`, `--force-checker`| Force usage of `check.cpp`. Error if missing.
| `-k`, `--force-diff` | Force standard `diff` comparison (ignore checker).
| `-u`, `--update` | Self-Update utility from GitHub.
|===

== Custom Checkers

For problems where the output is not unique, standard comparison fails. Create `check.cpp` and CppTest will automatically compile and use it to verify correctness.

**Signature:**
[source,cpp]
----
int main(int argc, char* argv[]) {
    ifstream inf(argv[1]); // Input
    ifstream ouf(argv[2]); // User Output
    ifstream ans(argv[3]); // Correct Output (if available)
    
    // Logic goes here...
    
    if (correct) return 0; // OK
    else return 1;         // Wrong Answer
}
----
